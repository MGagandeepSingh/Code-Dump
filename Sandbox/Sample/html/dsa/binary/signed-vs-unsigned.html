Signed v/s Unsigned

Updated: 21-Dec-2022
Author: Gagandeep Singh
Java uses signed data type, which means that the last bit's value determines if the number is positive or negative.
Since the last index is used to determine if the number is positive or not, the maximum value that can be stored in the remaining 7 bits is 127.

64 + 32 + 16 + 8 + 4 + 2 + 1

96 + 16 + 8 + 4 + 2 + 1

112 + 8 + 4 + 2 + 1

120 + 4 + 2 + 1 = 127

Positive numbers are stored in this way, but in order to store the negative numbers we make use of the 2's compliment. Let's see how -10 is stored in binary form in Java.
Steps to find the binary equivalent of -10
  1. Find the binary equivalent of 10
  2. Find 1's compliment. This is nothing but flipping all the digits from 0 to 1 and vice-versa
  3. Find 2's compliment. That is adding binary 1 to the last digit.
